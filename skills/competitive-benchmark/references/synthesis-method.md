# Synthesis Method — Steal / Differentiate / Avoid

Reference for the `competitive-benchmark` skill. This is where the matrix becomes strategy.
A benchmark that only describes is worthless; the value is the *direction*. The synthesis
must answer one question for the user: **"Given where everyone stands, what should I do?"**

---

## The Three Lenses

For each axis (or grouped where it reads more clearly), generate findings under three lenses.
Not every axis yields all three — force nothing.

### 🟢 STEAL — Adopt what a competitor does better
A competitor measurably outperforms the user on this axis, and the advantage is *adoptable*
without becoming a clone.

- **Name the competitor and the specific thing.** "Competitor B's pricing page leads with a
  3-tier comparison and an ROI line per tier" — not "B has better pricing."
- **Translate, don't transplant.** Say how to *adapt* it to the user's brand, not copy it
  pixel-for-pixel. The goal is the underlying principle, expressed in the user's own voice.
- **Only steal genuine strengths.** If it's a dark pattern, it goes to AVOID, not STEAL.

### 🔵 DIFFERENTIATE — Own the gap nobody occupies
Where every site in the set looks/sounds/behaves the same. **This is the highest-value lens
— bias the synthesis toward it.** Sameness in a crowded category is a liability; the
unoccupied position is the prize.

- **Identify the shared blind spot.** "All four sites use the same SaaS-blue, the same
  three-feature-card layout, the same 'Powerful. Simple. Fast.' triad." When everyone
  converges, convergence itself is the opening.
- **Propose the divergent move.** What could the user do that none of them do — in visual
  identity, voice, positioning, or experience — that would make them instantly distinct
  *and* still serve the user well? Differentiation for its own sake is noise; differentiation
  that's also *better for the customer* is strategy.
- **Tie it to a real user need the category ignores.** The best differentiation fills a gap
  customers feel but no competitor addresses.

### 🔴 AVOID — Don't inherit the category's mistakes
Things one or more competitors do badly, that the user should consciously *not* replicate.

- **Name the mistake and who makes it.** Clutter, dark patterns, weak/jargon positioning,
  broken journeys, slow load, trust-signal theatre.
- **Dark patterns always land here.** Fake countdowns, inflated "X people viewing", forced
  continuity, confirmshaming — flag as AVOID even if "effective." They erode trust and
  increasingly carry legal/reputational risk.
- **Watch for category-wide bad habits.** If everyone does the annoying thing (e.g. an
  immediate email-capture modal), *not* doing it is both an AVOID and a DIFFERENTIATE.

---

## The Cardinal Rule: Don't Build a Clone Factory

The failure mode of competitive analysis is a report that says "become more like the
leader." That produces a me-too product that competes only on execution — a losing game
against an incumbent with more resources.

The valuable synthesis does the opposite:
1. **Steal selectively** — only table-stakes things the user is genuinely behind on.
2. **Differentiate aggressively** — find the position no one owns and push the user into it.
3. **Avoid deliberately** — refuse the category's bad habits, especially the shared ones.

When in tension, **differentiation wins.** A site that's 80% as polished as the leader but
unmistakably *itself* beats a flawless clone.

---

## The One Move

Close every synthesis with a single, decisive recommendation: **if the user changed only
one thing, what would move the needle most?** This forces prioritisation and gives them
somewhere to start. It should usually be a DIFFERENTIATE play, occasionally a STEAL if the
user has a glaring table-stakes gap that's actively bleeding conversions.

State it as: **the move**, **why it's the highest leverage**, and **what it unlocks**.

---

## Tone of the Synthesis

- **Be decisive, not wishy-washy.** The user wants a direction, not seven hedged options.
- **Stay anchored to their business.** Every line answers "so what should *I* do?"
- **Honest about their strengths too.** If the user already leads on an axis, say so — tell
  them to protect and lean into it, don't fabricate problems.
- **No flattery, no doom.** A credible read is balanced: real gaps, real strengths, a clear
  path. Demoralising or sycophantic both kill trust in the analysis.
